--GitHub--

GitHub (filial de Microsoft) es un SCV en línea que almacena los archivos en la nube, debido a su facilidad y sencilla interfaz gráfica, GitHub es un usado por cientos de miles de usuarios para almacenar de forma pública (y privada) su contenido, tal es así, que a dia de hoy, diferentes consultoras como Everis y Accenture trabajan desde esta plataforma, permitiendo así que sus diferentes programadores puedan crear ramificaciones, al igual que en el propio GIT.

--Funcionamiento de GitHub--

GitHub funciona mediante la creación de repositorios en la nube, pero una de las funciones principales de GitHub, es la posibilidad de realizar ‘Forks’, es decir, realizar una copia local desde la cual puedes modificar los archivos, y luego, mandarlos al creador original, quien puede ignorarte, o añadirlos a su proyecto.

Si a uno de tus repositorios le realizan un fork, y desean actualizar tu repositorio, a ti como usuario se te genera un pull request, es decir, mergear su rama de trabajo, con la tuya.

--SCV--

Un Sistema de control de versiones es un sistema que registra los cambios realizados a un archivo, o conjunto de archivos, de modo que podamos recuperar versiones específicas más adelante.


Estos sistemas, nos permite regresar a versiones específicas de archivos, comparar cambios, ver quien y cuando realizo esos cambios.


Usar un Sistema de control de versiones nos permite recuperar archivos arruinados o borrados. En el caso de Git, de forma gratuita.


Con el paso del tiempo, se han creado los conocidos SCVD, Sistema de control de versiones distribuidas, en el que no solo se descarga la ultima copia instantánea de los archivos, sino que se replica completamente el repositorio. Gracias a esto, si un servidor deja de funcionar, gracias a este sistema, las personas que estaban colaborando con el mismo pueden ayudar a restaurar el propio servidor.
